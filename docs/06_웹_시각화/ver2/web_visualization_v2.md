# 06. 웹 시각화 상세 설계 v2

## 1. 개요

v1에서 v2로의 전면 리팩토링. Bloomberg Terminal 스타일 3-Column 레이아웃으로 전환하고, PixiJS를 제거하고 Canvas 2D 직접 렌더링으로 변경했다.

### 1.1 v1 대비 주요 변경사항

| 항목 | v1 | v2 |
|------|----|----|
| **레이아웃** | 오피스 Canvas + 사이드바 대시보드 | 3-Column (Agent Status / 탭 패널 / Activity Log) |
| **렌더링** | PixiJS 8 | Canvas 2D 직접 렌더링 |
| **사이드바 패널** | EDA, 피처, ML, 백테스트 등 7개 탭 | 시장 탭 + OFFICE 탭 (2개) |
| **오피스** | 캔버스 전체 꽉채움 | 중앙 탭 영역 내 렌더링 |
| **CEO실** | 없음 | 코리더 상단 독립 방 |
| **복도** | 단순 통로 | 센터라인, 천장등, EXIT 표지판 |
| **배달 시스템** | 랜덤 배달 | 파이프라인 순차 배달 (수집→분석→ML→보고서→CEO) |
| **테마** | 다크 모드 전용 | 다크/라이트 토글 |
| **차트** | 차트 라이브러리 의존 | Canvas 2D 캔들차트 직접 구현 |

### 1.2 제거된 패널

- `EDAPanel.tsx` — EDA 결과 패널
- `FeaturePanel.tsx` — 피처 중요도
- `MLModelPanel.tsx` — ML 모델 성능 비교
- `BacktestPanel.tsx` — 백테스트 결과
- `LossChart.tsx`, `ConfusionMatrix.tsx`, `PredictionChart.tsx`, `RiskGauge.tsx`
- In-canvas 정보 패널 (React 사이드바로 이동)

### 3.2 팀 방 공통 요소

각 팀 방은 아래 요소를 공유한다:

- **바닥**: 팀별 고유 색조 타일 + 대각선 카펫 텍스처
- **벽**: 5px 두께, 상단 액센트 스트립(팀 컬러 44%), 하단 걸레받이
- **창문**: 상단 벽에 2개, 유리 반사광 + 창틀
- **문**: 우측 벽 60% 높이에 22px 문 개구부 + 문 호(arc) 표시 + 프레임
- **방 번호**: 문 옆 R01~R04 배지
- **천장등**: 방 내부 2개, 팀 컬러 방사형 그라데이션 글로우
- **OUTBOX 트레이**: 문 옆 서류 적재대. 소형 테이블 위 적재 서류 + 최상단 서류에 팀 컬러 탭 + "OUT" 라벨. 서무 캐릭터의 출발/귀환 지점
- **팀 라벨 배지**: 좌상단 `[ COL ] COLLECTION` 형태

### 3.3 팀별 고유 가구

| 팀 | 고유 소품 |
|----|----------|
| 수집팀 (COL) | 파일링 캐비넷 (3단 서랍 + 금색 손잡이), 화분, 네트워크 스위치 (LED 6개) |
| 분석팀 (DSC) | 화이트보드 (차트 + "ADF✓" 텍스트), 책장 (5색 책), 화분 |
| ML팀 (ML) | GPU 서버 랙 (4유닛, 깜빡이는 LED, 활동 바), UPS 전원 스트립 |
| 보고서팀 (RPT) | 프린터 (용지 출력 애니메이션 + LED), 화분 |

### 3.4 CEO실

코리더 상단에 독립 배치. `ceoH = roomH * 0.7` 높이.

레이아웃 오프셋:
- `CEO_OFFSET = 16px` — 캔버스 상단으로부터의 여백
- `CEO_GAP = 14px` — CEO실과 복도 사이 간격

구성 요소:
- **바닥**: 다크 우드톤 + 수평 결 라인
- **벽**: 4면 + 하단 벽에 문 개구부 (금색 프레임)
- **금색 액센트 스트립**: 상단 벽 전체
- **라벨**: `[ CEO ] OFFICE`
- **이그제큐티브 데스크**: 중앙, 일반 데스크보다 크게(40%)
- **CEO 캐릭터**: 금색, `reviewing` 상태
- **명패**: 데스크 위 "CEO" 텍스트
- **책장**: 좌측 벽 전체 높이, 6색 책 스파인
- **트로피**: 우측 벽, 금색 원 + 받침대 + "MVP" 라벨
- **파노라마 창**: 상단 벽 중앙, 가로 30%

### 3.5 복도

CEO실 하단~캔버스 하단까지.

구성 요소:
- **바닥 타일**: 18px 격자
- **센터라인**: 중앙 수직 점선 (6px dash, 8px gap)
- **천장등 3개**: 등간격 배치, 직사각 피처 + 아래방향 방사형 글로우
- **EXIT 표지판**: 좌측 벽 4개 위치 (12%, 38%, 62%, 88%)에 녹색 EXIT 표시와 적색 소화기 마커를 번갈아 배치

### 3.6 파이프라인 배달 시스템

#### 순차 배달 경로

```
수집팀(R01) ──→ 분석팀(R02): "분석 데이터"    wait: 300틱
분석팀(R02) ──→ ML팀(R03):   "피처 세트"      wait: 350틱
ML팀(R03)  ──→ 보고서팀(R04): "모델 결과"     wait: 400틱
보고서팀(R04) ──→ CEO실:      "최종 보고서"    wait: 300틱
```

한 번에 하나의 배달만 활성화되며, 배달 완료 후 대기 시간(wait) 경과 시 다음 스테이지가 시작된다. 4단계 순환.

#### 서무(Delivery Staff) 캐릭터

각 팀 방에 전담 서무 캐릭터가 상주한다:

| 팀 | 캐릭터 ID | 위치 |
|----|-----------|------|
| 수집팀 | c6 | x:2, y:1 (OUTBOX 트레이/문 부근) |
| 분석팀 | a6 | x:2, y:1 |
| ML팀 | m5 | x:2, y:1 |
| 보고서팀 | r5 | x:2, y:1 |

- 역할: `'delivery'`, 평상시 IDLE 상태
- 배달 발생 시 OUTBOX에서 서류를 들고 출발, 도착 팀에 전달

#### 귀환 경로 (Return Trip)

배달 완료 후 서무 캐릭터는 원래 팀 방으로 복귀한다:

- `Delivery` 인터페이스에 `isReturn: boolean` 플래그 추가
- 귀환 시: 서류 미소지, 라벨 미표시
- 출발 팀 컬러 유니폼 유지
- 원래 팀 OUTBOX 트레이 도착 후 캐릭터 소멸

#### 5-Phase 이동 경로

```
Phase 1 (0~12%):  출발 팀 데스크 → 출발 팀 문
Phase 2 (12~22%): 출발 팀 문 → 복도 중앙
Phase 3 (22~72%): 복도 종방향 이동 (출발 문 Y → 도착 문 Y)
Phase 4 (72~82%): 복도 중앙 → 도착 팀 문
Phase 5 (82~100%): 도착 팀 문 → 도착 팀 데스크
```

CEO실 배달 시 Phase 4~5에서 복도→CEO 하단 문→CEO 데스크로 이동.

#### 일정 보행 속도

```typescript
const WALK_SPEED = 0.35  // pixels per tick
// progress 증분 = WALK_SPEED / totalDist
// totalDist = desk→door + door→corridor + corridor수직이동 + corridor→door + door→desk
```

총 거리(totalDist)를 사전 계산하여 `progress`가 거리에 비례하도록 함. 가까운 방은 빠르게, 먼 방은 오래 걸린다.

#### 배달 캐릭터 렌더링

- 바디 색상: 출발 팀 컬러 (수집=녹, 분석=시안, ML=보라, 보고서=금)
- 이동 방향에 따라 눈 위치 변경 (좌향/우향)
- 팀 컬러 모자
- 문서/폴더 아이콘 (이동 방향 반대측)
- 라벨: 출발 팀 컬러로 "분석 데이터", "피처 세트" 등 표시
- 보행 바운스: `sin(tick * 0.25) * 1.5`
